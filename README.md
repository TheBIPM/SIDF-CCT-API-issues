This repository is **issues-only**. It serves as a public tracker for bugs, questions and feature requests related to the [**CCT-API**](https://preprod.si-digital-framework.org/api-docs/swagger-ui/?urls.primaryName=TEST%3A%20CCT-API), which is part of the [**SI Digital Framework (SIDF)**](https://si-digital-framework.org/).
Please note that at the moment, the [**CCT-API**](https://preprod.si-digital-framework.org/api-docs/swagger-ui/?urls.primaryName=TEST%3A%20CCT-API) is only available to the members of CCT-WG-Dig.

## Scope
Use this repository exclusively for topics directly related to the [**CCT-API**](https://preprod.si-digital-framework.org/api-docs/swagger-ui/?urls.primaryName=TEST%3A%20CCT-API), for example:
- Missing identifiers in the API outputs.
- Questions about endpoint behaviour, parameters, or response formats.

For issues about other SIDF components, please use the corresponding repository or send us an email.

## How to create an issue
1. Search existing issues:
   Check the open and closed issues to see if your problem or request has already been reported or discussed. In such case add your comments to the discussion.

2. If your request has not been discussed, choose the type of issue that better fits the situation you want to report.
   - `Bug report`
   - `Feature request`
   - `Question / clarification`

3. When filling the issue, please include:
  - For bugs: expected behaviour vs actual behaviour.
  - For feature requests: what you need, why, and any constraints.
  - For questions: a concrete example call or use case.
